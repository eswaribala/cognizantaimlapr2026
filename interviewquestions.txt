1. Set and Frozen Set
A frozenset in Python is an immutable version of a set.

Normal set → can add/remove elements
frozenset → cannot be modified after creation

Syntax
fs = frozenset(iterable)

Example:

numbers = frozenset([1, 2, 3, 4])

print(numbers)

Output:

frozenset({1, 2, 3, 4})
Why use frozenset?
Immutable (read-only)
Can be used as a dictionary key

Dynamic pivot for unknown cities
SET @sql = NULL;

SELECT GROUP_CONCAT(
    DISTINCT CONCAT(
        'SUM(CASE WHEN city = ''',

FROM customers;

SET @sql = CONCAT('SELECT ', @sql, ' FROM customers');

DEALLOCATE PREPARE stmt;

3. Moving Average
A moving average is a technique used to smooth data by taking the average of a fixed number of recent values.

It is commonly used in:

Finance (stock prices)
Time series analysis
Sales trends
Sensor/IoT data
Machine learning preprocessing
Formula

For a 3-period moving average:


MA=x1​+x2​+x3​/3
	​


It means: take the last 3 values, add them, divide by 3.

Example

Suppose sales data:

Day	Sales
1	10
2	20
3	30
4	40
5	50

For 3-day moving average:

Day 3:

(10 + 20 + 30) / 3 = 20

Day 4:

(20 + 30 + 40) / 3 = 30

Day 5:

(30 + 40 + 50) / 3 = 40

Result:

Day	Sales	Moving Avg (3)
1	10	NULL
2	20	NULL
3	30	20
4	40	30
5	50	40
Python Example
sales = [10, 20, 30, 40, 50]

window = 3

moving_avg = []

for i in range(len(sales)):
    if i < window - 1:
        moving_avg.append(None)
    else:
        avg = sum(sales[i-window+1:i+1]) / window
        moving_avg.append(avg)

print(moving_avg)

Output:

[None, None, 20.0, 30.0, 40.0]
Using Pandas

pandas

import pandas as pd

df = pd.DataFrame({
    "sales": [10, 20, 30, 40, 50]
})

df["moving_avg"] = df["sales"].rolling(window=3).mean()

print(df)

Output:

   sales  moving_avg
0     10         NaN
1     20         NaN
2     30        20.0
3     40        30.0
4     50        40.0
Moving Average in SQL (MySQL)

Using window functions:

SELECT
    day,
    sales,
    AVG(sales) OVER (
        ORDER BY day
        ROWS BETWEEN 2 PRECEDING AND CURRENT ROW
    ) AS moving_average
FROM sales_table;

This calculates a 3-row moving average.

Example:

Day | Sales | Moving Avg
1   | 10    | 10
2   | 20    | 15
3   | 30    | 20
4   | 40    | 30
5   | 50    | 40

Difference:

Simple Average → average of all values
Moving Average → average of recent values only (rolling window)

4. Window functions in SQL perform calculations across a set of rows related to the current row without grouping rows into one result.

Unlike GROUP BY, window functions keep individual rows and add calculated values.

Syntax
FUNCTION() OVER (
    PARTITION BY column
    ORDER BY column
)
OVER() → makes it a window function
PARTITION BY → divides rows into groups (optional)
ORDER BY → defines order inside group
ROWS/RANGE → defines window frame
Why use window functions?

Instead of:

SELECT department, AVG(salary)
FROM employee
GROUP BY department;

which gives one row per department,

window function gives:

SELECT
    name,
    department,
    salary,
    AVG(salary) OVER(PARTITION BY department) AS dept_avg
FROM employee;

Output:

name	department	salary	dept_avg
Ravi	IT	50000	60000
Priya	IT	70000	60000
Arun	HR	40000	45000
Meena	HR	50000	45000

Each row remains.

employee

2  | Priya | IT   | 70000
3  | Arun  | HR   | 40000
4  | Meena | HR   | 50000

2. Implement LRU Cache

Scenario:
API responses are expensive. Cache only recent items.

from collections import OrderedDict

class LRUCache:
    def __init__(self, capacity):
        self.capacity = capacity
        self.cache = OrderedDict()

    def get(self, key):
        if key not in self.cache:
            return -1

        self.cache.move_to_end(key)
        return self.cache[key]

    def put(self, key, value):
        if key in self.cache:
            self.cache.move_to_end(key)

        self.cache[key] = value

        if len(self.cache) > self.capacity:
            self.cache.popitem(last=False)


cache = LRUCache(3)

cache.put(1, "A")
cache.put(2, "B")
cache.put(3, "C")
cache.put(4, "D")

print(cache.cache)

3. Thread-safe Singleton

import threading

class Singleton:
    _instance = None
    _lock = threading.Lock()

    def __new__(cls):
        with cls._lock:
            if cls._instance is None:
                cls._instance = super().__new__(cls)

        return cls._instance

6. Retry Decorator

Scenario:
Retry flaky API.

import time

def retry(times):

    def decorator(func):

        def wrapper(*args, **kwargs):

            for i in range(times):
                try:
                    return func(*args, **kwargs)
                except Exception:
                    time.sleep(1)

            raise Exception("Failed")

        return wrapper

    return decorator


@retry(3)
def api():
    print("Calling API")
    raise Exception("Network error")


api()

7. Rate Limiter

Scenario:
Only 5 API calls/sec.

import time

class RateLimiter:

    def __init__(self, limit):
        self.limit = limit
        self.calls = []

    def allow(self):
        now = time.time()

        self.calls = [
            x for x in self.calls
            if now - x < 1
        ]

        if len(self.calls) < self.limit:
            self.calls.append(now)
            return True

        return False

9. Detect Circular Dependency

Scenario: Graph problem

graph = {
    "A": ["B"],
    "B": ["C"],
    "C": ["A"]
}

visited = set()

def dfs(node, stack):
    if node in stack:
        return True

    if node in visited:
        return False

    visited.add(node)
    stack.add(node)

    for neighbor in graph[node]:
        if dfs(neighbor, stack):
            return True

    stack.remove(node)

    return False


print(dfs("A", set()))

Types of MySQL Indexes
Index Type	Purpose
Primary Index	Unique identifier for a table
Unique Index	Prevent duplicate values
Normal (Simple) Index	Faster search
Composite Index	Multiple columns
Full-Text Index	Text searching
Spatial Index	GIS/location data
Clustered Index	Physical row storage order
Non-Clustered Index	Separate lookup structure
Covering Index	Query satisfied from index alone
1. Primary Index (PRIMARY KEY)

Used to uniquely identify a row.

Properties:

Unique
Cannot be NULL
One per table

Table:

CREATE TABLE employee (
    id INT PRIMARY KEY,
    name VARCHAR(100),
    salary DECIMAL(10,2)
);

Query:

SELECT *
FROM employee
WHERE id = 101;

Fast lookup.

Think: Aadhaar number / employee ID.

2. Unique Index

Prevents duplicates.

Example:

CREATE UNIQUE INDEX idx_email
ON customers(email);

Now duplicate email not allowed.

Example:

INSERT INTO customers(email)
VALUES('abc@gmail.com');

Duplicate insert:

Duplicate entry error

Use case:

Email
PAN
Passport number
3. Normal (Simple) Index

Improves search performance.

CREATE INDEX idx_name
ON employee(name);

Query:

SELECT *
FROM employee
WHERE name='Ravi';

Without index:

Full table scan

With index:

Fast search
4. Composite Index (Multi-column)

Index on multiple columns.

Example:

CREATE INDEX idx_city_salary
ON employee(city, salary);

Works well for:

SELECT *
FROM employee
WHERE city='Chennai'
AND salary > 50000;

Also works:

WHERE city='Chennai'

But not efficient:

Because of leftmost prefix rule.

Example:

(city, salary)

Efficient:

city
city + salary

Not efficient:

salary only
